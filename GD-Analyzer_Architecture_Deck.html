GD Analyzer

A browser-based simulator for distributed-generation (GD) energy savings — how it's built, the systems behind it, and why it now runs on Vercel instead of GitHub Pages.

React 19 + TypeScript Vite 8 ANEEL open data Deployed on Vercel
What the app does

A self-contained GD savings simulator

  • Imports client electricity bills & consumption (PDF / Excel / CSV)
  • Pulls real distributor tariffs from ANEEL (TUSD, TE, taxes per state)
  • Simulates GD compensation & energy credit banking
  • Optimises demand contracts & allocation
  • Exports polished Excel & PDF proposals

Key traits

  • Offline-first — runs entirely in the browser
  • No backend server — projects saved locally in IndexedDB
  • Only external call: ANEEL tariff API
  • Heavy math runs in a Web Worker (UI stays smooth)

The hosting challenge

Why "refresh tariffs" can't run on a static host

  • The browser asks ANEEL for tariffs directly
  • ANEEL replies — but sends no CORS header
  • So the browser blocks the response for safety
  • The fix is a server in the middle to relay the call
  • GitHub Pages has no server — it only serves files

The 3 transports the app tries

  • /api/aneel/… ✗ 404 on a static host
  • Direct to ANEEL ✗ blocked by CORS
  • corsproxy.io ✗ now returns 403 (dead)

On GitHub Pages all three fail → button silently falls back to the bundled file.

GitHub Pages vs Vercel

Same code, different powers

CapabilityGitHub PagesVercel
Serve the built app (HTML/JS/CSS)✓✓
Free for personal use✓✓
Run server-side logic / proxy a request✗ files only✓ rewrites & functions
Make the ANEEL "refresh" button work live✗ CORS-blocked✓ same-origin proxy
Auto-deploy on git pushvia action✓ built-in

GitHub Pages is a file cabinet. Vercel is a file cabinet with a receptionist who can fetch things for you.

What Vercel adds

One rewrite rule solves the CORS problem

// vercel.json — runs on Vercel's server, not the browser { "rewrites": [ { "source": "/api/aneel/:path*", "destination": "https://dadosabertos.aneel.gov.br/api/3/action/:path*" } ] }
  • Browser calls /api/aneel/… — same origin, so no CORS
  • Vercel's server fetches ANEEL and pipes the answer back
  • It reproduces the local Vite dev proxy — in production
  • Zero app-code changes — the existing fetch path just works

Decision

Can we remove it from GitHub Pages?

Yes

Vercel now does everything Pages did — plus the working proxy. The Pages copy has a permanently broken refresh button.

What "remove" means

  • Delete the gh-pages branch
  • Turn Pages off in repo Settings
  • Source code stays in the repo
  • Only the old github.io URL goes 404

Nothing important is deleted — just one redundant hosting endpoint.
